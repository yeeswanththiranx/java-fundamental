Primary Requirement: Platform
What is a Platform?
A platform is a base or environment where something can operate, run, or be built upon. It comprises an Operating System (Hardware Manager) and its associated hardware.
Examples of Platforms
  • Windows – Microsoft's OS family
  • macOS – Apple's desktop OS
  • Linux – Open-source UNIX-like OS
  • Android – Google's mobile OS
What a Platform Includes
  • Operating System (hardware manager)
  • CPU architecture (x86, ARM…)
  • Memory & storage management
  • Device drivers & I/O interfaces

Java Toolkit
JDK vs JRE
JDK – Java Development Kit
Required to develop & run Java programs. Contains the compiler (javac), debugger, and the JRE.
  • Includes javac (compiler)
  • Includes the Java debugger
  • Includes the complete JRE
JRE – Java Runtime Environment
Required only to run already-compiled Java programs. Does not include the compiler.
  • Includes the JVM (Java Virtual Machine)
  • Includes core Java class libraries
  • No compilation tools
⚠️ Rule of thumb: Developers install JDK. End users who only run apps can install JRE.

Java Syntax
Syntax for a Java Class
Every Java program starts with at least one class definition:
Java 
class ClassName {
    // fields, methods, constructors go here
}
  • The keyword class declares a new class.
  • ClassName follows PascalCase convention (e.g., HelloWorld).
  • Curly braces { } wrap the entire class body.
  • The file name must match the public class name exactly.

Execution
Why Is main() Required?
The main() method tells the JVM where to start. Without it, the program has no entry point to run.
  • Execution begins at the first line of main()
  • Execution ends at the closing brace of main()
  • Without it: compiles fine, cannot run
Java 
public class Hello {
  // No main method here!
  public void greet() {
    System.out.println("Hello");
  }
}
❌ Compiles OK. Running gives:
Error: Main method not found in class Hello

Slide 9
Flexibility
Can You Rename args?
Yes! The JVM only cares about the type String[] — not the parameter name. args is just a convention.
Java — all three are valid 
public static void main(String[] args)   { } // conventional
public static void main(String[] myArgs) { } // custom name
public static void main(String[] input)  { } // also valid
✅ The parameter name can be anything. The parameter type (String[]) is mandatory and cannot change.

Slide 10
Overloading
Overloading the main() Method
Java 
public class Example {
  public static void main(int[] args) {  // overloaded version
    System.out.println("This is an overloaded main.");
  }
  public static void main(String[] args) { // JVM entry point
    Example obj = new Example();
  }
}
✅ JVM always calls main(String[]) as the entry point. Other overloads are ignored unless explicitly called.
⚠️ Overloading main() is technically allowed but is considered bad practice. Avoid it in production code.

5 Rules for the main() Method
Rule 1
Must be public
Rule 2
Must be static
Rule 3
Return type must be void
Rule 4
Name must be lowercase main

Parameter must be String[]

Java Trivia
Important Miscellaneous Rules
  • Package statement comes first.
    The package statement, if present, must be the very first line in a Java source file — before any imports or class declarations.
  • Whole numbers default to int.
    When you write 42 in Java, the compiler treats it as an int by default. Use 42L for long.
  • Fractional numbers default to double.
    Writing 3.14 gives you a double. Use 3.14f for a float literal.
Java
int    a = 42;       // default: int
long   b = 42L;      // explicit long
double c = 3.14;     // default: double
float  d = 3.14f;    // explicit float
